 Without MainController:
hey from message3

Reason:
Because getMessage2() depends on getMessage3(), Spring creates getMessage3() first, then getMessage2().
getMessage1() has no dependencies, so it is created afterward.
Therefore, the execution order follows the dependency chain.

With MainController:Error
Reason:
Because getMessage1() depends on MainController, and MainController depends on getMessage2(), and getMessage2() depends on getMessage3().
This creates a dependency chain that results in a circular dependency, so the application fails to start.

Reason:
Because MainController depends on getMessage2() using @Qualifier("2"), and getMessage2() depends on getMessage3() using @Qualifier("3").
